GPU Architectural Shift: Solving JIT Crashes

Moving from a monolithic "Monstrosity" kernel to a stable, GPU-native 3-stage pipeline.

CURRENT: Monolithic Kernel (CRASHING)

Single Kernel Thread

  • Loop: 5 centers
  • Loop: 1000s of stocks
  • Loop: Start times
  • Loop: Search times
  • Loop: Select times
  • Deep Nested Logic <-- JIT fails here
  • Manual loop counter updates
  • Complex branching (break/continue)
PROPOSED: Triple-Kernel Pipeline (STABLE)

1. Indicators Kernel

Pre-computes MA Gap, RVol, etc.

⬇️

2. Heuristic Scorer Kernel

Parallel across Centers × Stocks × Days.

(NO LOOPS! Pure Math)

⬇️

3. Simulation Runner Kernel

Simple scan through pre-computed scores.

Why this works:

  • ✅ Zero Nesting: Most work happens in Kernel 2 with 0 loops.
  • ✅ GPU Optimized: Kernel 2 uses 100% of GPU compute cores simultaneously.
  • ✅ Stable Syntax: Removes all `break`, `continue`, and `i=curr` patterns.
